Provides classes to demonstrate opb-plsql use in a JSE environment.

Database set-up

To run all of the demo's you'll need to create 2 database (DB) accounts: opb_jse_demo and opb_jse_light_demo.

opb_jse_demo

The opb_jse_demo account is set-up with all Opb features enabled (such as session tracking and the ability to create user messages).
The opb_jse_light_demo account is used by SimpleArrayDemoMain, SimpleBooleanDemoMain, SimpleMessagesDemoMain and USerDefinedCollectionDemoMain.
  1. Create the DB account (opb_jse_demo). e.g. Run the following commands via SQL*Plus:
    • CREATE USER opb_jse_demo DEFAULT TABLESPACE users IDENTIFIED BY weak_pw;
    • GRANT DBA TO opb_jse_demo;
  2. Build the Opb SQL:
    • Run src/main/resources/sql/build.sql from the opb-plsql project as opb_jse_demo.
  3. Build the demo SQL
    • Run src/main/resources/sql/build.sql from this project as opb_jse_demo.
The default DB connection details are (see applicationContext.xml);
  • url=jdbc:oracle:thin:@//localhost:1521/xe
  • user=opb_jse_demo
  • password=weak_pw
This URL will work if you have a local copy of Oracle XE running.
If you would like to use a different URL or user, please edit the configuration of the oracleDataSource bean in src/main/resources/applicationContext.xml.

opb_jse_light_demo

The opb_jse_light_demo account could be any DB account and does not need Opb PL/SQL objects to be compiled.
The opb_jse_light_demo account is used by LightSessionDemoMain.
  1. Create the DB account (opb_jse_light_demo). e.g. Run the following commands via SQL*Plus:
    • CREATE USER opb_jse_light_demo DEFAULT TABLESPACE users IDENTIFIED BY weak_pw;
    • GRANT DBA TO opb_jse_light_demo;
The default DB connection details are (see applicationContext.xml);
  • url=jdbc:oracle:thin:@//localhost:1521/xe
  • user=opb_jse_light_demo
  • password=weak_pw
This URL will work if you have a local copy of Oracle XE running.
If you would like to use a different URL or user, please edit the configuration of the oracleDataSource bean in src/main/resources/applicationContext.xml.
